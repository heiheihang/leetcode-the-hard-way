Time Complexity

This tutorial is written by@wingkwong

Overview​

Time Complexity is one of the important measurements when it comes to writing an efficient solution. It estimates how much time your solution needs based on some input. If your solution is too slow, even it passes some test cases, it will still consider it as a wrong answer.

The time complexity is denoted by Big O notation. Normally, nnn means the input size. For example, if the input is a string sss, then nnn would be the length of sss.

Example 1:​

In the following case, the time complexity depends on nnn. Therefore, it is O(n)O(n)O(n).

for (int i = 0; i < n; i++) {
  // do something
}

Example 2:​

In the following case, the time complexity depends on nnn and mmm. Therefore, it is O(n∗m)O(n*m)O(n∗m).

for (int i = 0; i < n; i++) {
  for (int j = 0; j < m; j++) {
    // do something
  }
}

Example 3:​

In the following case, the time complexity is O(n)O(\sqrt n)O(n​). You can see i∗i<=ni * i <= ni∗i<=n as i<=ni <= \sqrt ni<=n​.

As sqrt() returns double, it would be safe to use i * i <= n to check the condition instead of using i <= sqrt(n).

for (int i = 2; i * i <= n; i++) {
  // do something
}
