Binary Exponentiation

Author:@wingkwong

Binary Exponentiation is also known as Exponentiation by Squaring or even called Fast Pow in some CP templates. It is used to calculate ana ^ nan using only O(logn)O(logn)O(logn) multiplications instead of O(n)O(n)O(n)using naive approach.

We can express ana ^ nanas a∗a∗...∗aa * a * ... * aa∗a∗...∗a but it is not efficient for a large aaa and nnn. If we display the exponent in binary representation, says 13 = 1101, then we have 313=31101=38∗34∗31.3 ^{13} = 3^{1101} = 3^8*3^4*3^1.313=31101=38∗34∗31. Supposing we have a sequence a1,a2,a4,...,a⌊log2n⌋a ^ 1, a ^ 2, a ^4, ..., a^{\lfloor log_2 n\rfloor}a1,a2,a4,...,a⌊log2​n⌋, we can see the an element in the sequence is the square of previous element, i.e. 34=(32)23 ^ 4 = (3^2)^234=(32)2. Therefore, to calculate 3133 ^ {13}313, we just need to calculate ⌊log213⌋=3{\lfloor log_2 13\rfloor} = 3⌊log2​13⌋=3 times, i.e. (1 -> 4 -> 8). We skip 2 here because the bit is not set. This approach gives us O(logn)O(log n)O(logn) complexity.

To generalise it, for a positive integer nnn, we have

C++ Implementation​

long long fastpow(long long base, long long exp) {
  long long res = 1;
  while (exp > 0) {
    // if n is odd, a ^ n can be seen as a ^ (n / 2) * a ^ (n / 2) * a
    if (exp & 1) res *= base;
    // if n is even, a ^ n can be seen as a ^ (n / 2) * a ^ (n / 2)
    base *= base;
    exp >>= 1;
  }
  return res;
}

In case you need to take mod.

long long modpow(long long base, long long exp, long long mod) {
  base %= mod;
  long long res = 1;
  while (exp > 0) {
    if (exp & 1) res = (res * base) % mod;
    base = (base * base) % mod;
    exp >>= 1;
  }
  return res;
}
